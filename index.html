Why Dashpipe

Four pillars. One platform.

From visual design to AI-assisted development, containerized execution, and agentic operations — without heavyweight orchestration frameworks.

01

No-code data orchestration

Design linear Source → Transform → Destination pipelines in a visual builder — no YAML, no DAG wiring.

  • Drag-and-drop pipeline builder with connector wizards
  • Import/export portable JSON bundles across tenants
  • Reusable connector SPI — REST, storage, message bus
  • Multi-tenant control plane with draft → active lifecycle

React UI · Spring Boot API

02

Agent AI pipeline development

Describe intent in natural language — get proposed pipelet steps, import to the platform, and scaffold new pipelets with an LLM.

  • Pipeline Guide — chat-driven design with live pipelet catalog
  • Iterative refinement — edit steps across follow-up messages
  • One-click import — bundle → draft pipeline on the platform
  • Pipelet Developer — Python + Dockerfile + K8s Job YAML

Streamlit · LangGraph · Ollama or cloud LLMs

03

On-demand scalable runtime

Each pipeline stage runs as its own containerized pipelet Job — spin up when needed, scale with workload, tear down when done.

  • One Kubernetes Job per pipelet stage — no shared worker pool
  • Catalog of sources, processors, and destinations
  • Message broker handoff between stages (RabbitMQ / Service Bus)
  • Local Compose dev or AKS production-style runs

K8s Jobs · pipelet containers · broker queue

04

Agent AI platform ops

Operate tenants, pipelines, executions, and connectors with an AI agent — dry-run before run, debug on failure, no API memorization.

  • Ops chat agent — LangGraph ReAct with 30+ platform tools
  • Full ops panel — pipelines, executions, connectors, catalog
  • Debug loop — dry-run → run → debug_execution on failure
  • MCP integrations — Cursor, Claude Desktop, OpenClaw, GitHub Copilot / VS Code

Shared in-process tools · MCP for Cursor, Claude, OpenClaw, Copilot

How the four pillars connect

Design in the no-code builder or AI dev agent. The control plane validates and schedules pipelet container Jobs. The ops agent monitors, runs, and debugs executions end to end.

  • 01 Orchestration — React builder, connectors, tenant-scoped pipelines
  • 02 AI development — Streamlit agents, LangGraph, import bundles
  • 03 Runtime — K8s pipelet Jobs, broker queues, on-demand containers
  • 04 AI ops — Platform Ops agent, MCP, observability & debug tools

Run all four pillars locally

Platform stack first (no-code + runtime), then the AI agent stack (development + ops).

# 01 & 03 — Platform: no-code builder + container runtime
./dashpipe-ci_cd/scripts/localdev.sh start --with-metrics

# 02 & 04 — AI agents: pipeline development + platform ops
./dashpipe-dev-ai-agent/scripts/localdev-ai.sh start

# Real K8s pipelet Jobs (on-demand runtime)
./dashpipe-ci_cd/scripts/localdev.sh start --k8s

# End-to-end demo: S3 → Petstore inventory
./dashpipe-ci_cd/scripts/inventory-pipeline-e2e.sh
