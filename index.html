Built for platform teams and data engineers

Repeatable, tenant-scoped pipelines you can share, meter, and operate — not one-off scripts.

Pipeline builder

Drag-and-drop linear pipelines, connector wizards, and import/export of JSON bundles.

Pipelet model

Each step is a containerized job with a catalog of sources, transforms, and destinations.

Connector SPI

Reusable REST, storage, and message-bus connectors scoped per tenant.

Observability

Prometheus metrics, Grafana dashboards, execution logs, and debug panels in the UI.

Local & cloud

RabbitMQ + MySQL + K8s locally, or AKS + Azure Service Bus for production-style runs.

AI & MCP

Optional LangGraph agents and an MCP server for pipeline design and live debugging in Cursor.

Control plane meets Kubernetes execution

The React UI talks to a Spring Boot API that orchestrates pipelet Jobs, tracks executions, and integrates with your message broker and MySQL metadata store.

  • Presentation — Pipeline builder, observability panels
  • Control plane — Tenants, pipelines, connectors, billing hooks
  • Execution — K8s Jobs per pipelet stage
  • Infrastructure — RabbitMQ or Service Bus, MySQL, Prometheus

Run locally in minutes

Prerequisites: Docker, Java 21, Node 20. Optional: a local Kubernetes cluster for real pipelet Jobs.

# Platform stack (Compose + API + UI)
./dashpipe-ci_cd/scripts/localdev.sh start --with-metrics

# Optional: real K8s pipelet runs
./dashpipe-ci_cd/scripts/localdev.sh start --k8s

# Flagship demo: S3 → Petstore inventory
./dashpipe-ci_cd/scripts/inventory-pipeline-e2e.sh
