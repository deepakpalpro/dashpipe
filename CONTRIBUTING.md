## Before you open a PR

1. Fork the repository and create a feature branch from `main`.
2. Run relevant tests for the component you changed (see [TESTING.md](docs/contributing/TESTING.md)).
3. Update documentation if you change user-facing behavior or paths.
4. Fill out the pull request template completely.
